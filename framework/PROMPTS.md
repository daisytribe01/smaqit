# Prompts

Prompts are the recorded invitations that start every workflow. They capture requirements, preferences, and constraints in natural language and serve as enduring input records.

## Key Principles

Prompts are treated as authoritative statements of intent, written in free form and enriched by gentle examples kept separate from the content. Each layer maintains a single prompt that accumulates evolving understanding.

## Prompts as Input Records

Captured prompts make the system reproducible: the same collection of prompts yields the same expected behavior. Edits to prompts make changes explicit and visible.

## Prompt Structure

Light structure and placeholders organize thought without enforcing rigid forms. Comments hold illustrations so real requirements stay clean.

## Agent Interaction

Agents read prompts as their source of truth, disregard commentary, and ask for clarity when meaning is unclear. Ambiguity is surfaced early rather than hidden.

## Amendment Rhythm

When intent shifts, prompts are updated first, and downstream artifacts realign accordingly. The prompt remains the anchor for what was requested.

## Prompt Types

Layer prompts capture intent, behavior, technology, environment, and verification perspectives. Phase and orchestration prompts capture how and when to run the work while honoring the same principles of clarity and traceability.
