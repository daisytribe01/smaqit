# Artifacts

Artifacts are the tangible expressions of intent within smaqit. Specification artifacts declare desired truths; implementation artifacts describe realized behavior. Together they preserve lineage from request to result.

## Specification Artifacts

Specifications act as contracts and as vehicles for shared understanding. They carry unique identifiers (e.g., `[LAYER_PREFIX]-[CONCEPT]-[NNN]`) for lineage, keep acceptance ideas testable, and reference their origins to preserve traceability. Foundations capture shared ideas that many features rely on, while feature expressions address specific outcomes. Completeness means the story can be implemented or validated without guessing, while remaining free of implementation detail.

## Traceability

References connect requirements, context, and verification so impact analysis and coverage remain clear. Reference patterns distinguish one-to-one intent from shared foundations without turning context into a second source of requirements.

## Implementation Artifacts

Implementation expresses behavior while respecting the boundaries set by specifications and recognized standards. Behavior stays invariant, structure remains recognizable, and internal choices may vary. Isolation keeps sensitive material outside the agent's reach, relying on trusted layers for resolution. Traceability markings keep validation possible without dictating exact code shape.

## Validation

Behavior is assessed through observable outcomes, structural expectations lean on recognizable standards, and internal variation is tolerated so long as the contract holds.
