# Session: Installer Refinements & Structure Changes

**Date:** 2025-12-19

## Summary

Refinement session focused on installer improvements and project structure rationalization. Clarified Level 0/1/2 boundaries, moved specs to project root, removed agent templates from installation, added smaqit. prefix to prompts, and created task for framework bundling investigation.

## Actions Taken

### Installer Enhancements

**make clean improvements:**
- Added `test/` directory cleanup to Makefile
- Prevents test artifacts from accumulating in installer directory

**smaqit init improvements:**
- Added optional directory parameter: `smaqit init [dir]`
- Creates target directory if it doesn't exist
- Changes to target directory before scaffolding
- Enables clean project initialization: `smaqit init my-project`

**Build system improvements:**
- Added `prepare` target to Makefile
- Automatically copies embedded files (framework/, templates/specs/, agents/) before build
- Resolves go:embed compile-time requirement
- Build now depends on prepare: `make build` → `prepare` → compile

### Level Boundaries Clarification

Established clear separation between development levels:

**Level 0 (smaqit source):**
- Framework files (6 .md files) — design docs for smaqit developers
- Templates for creating agents — used to build Level 2 agents
- Not shipped to users

**Level 1 (templates):**
- Spec templates (`templates/specs/*.template.md`) — used by agents at runtime
- Shipped to users in `.smaqit/templates/specs/`

**Level 2 (user artifacts):**
- Agent definitions (`.github/agents/*.agent.md`) — shipped to users
- Copilot prompts (`.github/prompts/*.prompt.md`) — shipped to users (when implemented)
- Specs produced by agents (`specs/`) — generated by user

**Installer changes:**
- Removed `.smaqit/templates/agents/` from installation
- Only ship spec templates (needed by agents)
- Agent templates remain in smaqit repo only

### Project Structure Changes

**Moved specs/ to project root:**
- Changed from `.smaqit/specs/` to `specs/`
- Rationale: Specs are Level 2 artifacts (user-generated), not framework files
- Updated all agent Output locations to `specs/{layer}/`
- Updated installer directory creation and validation
- Updated framework documentation references

**New user project structure:**
```
user-project/
├── .smaqit/
│   ├── framework/           # Reference docs (Level 0)
│   └── templates/specs/     # Spec templates (Level 1)
├── specs/                   # User specs (Level 2)
│   ├── business/
│   ├── functional/
│   ├── stack/
│   ├── infrastructure/
│   └── coverage/
└── .github/
    └── agents/              # Agent definitions (Level 2)
```

## Decisions Made

| Decision | Rationale |
|----------|-----------|
| Remove agent templates from installation | Only used for smaqit development (Level 0), not needed by users |
| Move specs/ to project root | Specs are user artifacts, not framework files; clearer separation |
| Add smaqit. prefix to prompts | Namespace separation; clearer branding; collision prevention |
| Add optional directory to init | Enables clean project initialization without manual mkdir |
| Make build depends on prepare | Ensures embedded files exist before compilation |
| Investigate framework bundling | Question whether full Level 0 docs should ship to Level 2 users |

## Technical Notes

**go:embed compile-time requirement:**
- Embedded files MUST exist in same module at build time
- Cannot use `../` paths in go:embed directives
- Solution: Copy files to installer/ before build (gitignored)
- `make prepare` handles copying; `make clean` removes copies

**Level boundaries:**
- Level 0: smaqit developers (this repo)
- Level 1: Template creators (spec templates shipped)
- Level 2: End users (agents, prompts, generated specs)

**Layer Independence reminder:**
- Each layer receives user input directly
- Upstream specs provide context, not requirements
- ARTIFACTS.md Purpose section now aligns with this principle
