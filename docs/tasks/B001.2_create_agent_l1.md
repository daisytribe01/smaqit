# Create Agent-L1: Template Compiler

**Status:** Completed  
**Created:** 2026-01-11  
**Completed:** 2026-01-11  
**Parent:** B001 (Extensible Meta-Framework: Level Up Architecture)  
**Priority:** High

## Goal

Create the **forward-looking** Level 1 Template Compiler agent (`smaqit.L1`) that compiles Level 0 principles into Level 1 template directives. This agent maintains template purity by ensuring directives are properly compiled from L0 principles.

## Context

**What this is NOT:** A retroactive template cleanup agent

**What this IS:** A compilation agent that:
- Compiles L0 principles into L1 directives
- Validates new template directives trace to L0 principles
- Maintains template structure and placeholder consistency
- Guides users on what belongs at L1 vs L0/L2

**Philosophy:** The agent assumes Level 1 templates are properly structured and maintains compilation discipline going forward through evolutionary development.

## Agent Responsibilities

### Input Scenarios

**Scenario 1: Compile New Principle to Directive**
- User: "Compile 'Single Source of Truth' principle to template directives"
- Agent: Generates MUST/SHOULD/MUST NOT directives for appropriate templates

**Scenario 2: Enhance Template with Missing Directives**
- User: "Add directives for Layer Independence to specification template"
- Agent: Compiles L0 principle into template-appropriate MUST/MUST NOT statements

**Scenario 3: L0 Contamination Prevention**
- User: "Add to template: Layer Independence means each layer receives requirements from its prompt"
- Agent: "This is a principle (L0), not a directive. The compiled directive would be: 'MUST read from layer prompt file only'"

**Scenario 4: L2 Contamination Prevention**
- User: "Add to template: Read from `.github/prompts/smaqit.business.prompt.md`"
- Agent: "This is L2 (concrete path). The L1 directive uses placeholder: 'Read from .github/prompts/smaqit.[LAYER].prompt.md'"

### Core Behaviors

**Agent MUST:**
- Accept input about template directives (additions, enhancements, clarifications)
- Validate directives compile from L0 principles
- Reject principle-form input (redirect to Agent-L0)
- Reject concrete implementations (redirect to Agent-L2)
- Maintain placeholder structure ([LAYER], [CONCEPT], [PREFIX])
- Preserve template structure and consistency
- Guide users when they provide L0 philosophy or L2 concrete content

**Agent MUST NOT:**
- Accept narrative philosophy without directives (that's L0)
- Accept concrete values without placeholders (that's L2)
- Modify L0 framework files
- Modify L2 agents
- Add specific examples (BUS-LOGIN-001, JWT, authentication)

**Agent SHOULD:**
- Trace directives back to L0 principles
- Suggest which L0 principle a directive implements
- Flag when directive has no clear L0 principle source
- Maintain consistent directive language across templates
- Use appropriate placeholder format

**File:** `.github/agents/smaqit.L1.agent.md`

**Sections:**

- Constraints:
  - Scope Boundaries (L1 only, no L0/L2 modification)

  - Failure Handling (error scenarios)
- Directive Form Guidance — Examples of proper L1 directives vs L0/L2

## Key Distinctions

| Aspect | Agent-L0 | Agent-L1 (this task) | Agent-L2 |
|--------|----------|----------------------|----------|
| Level | Principles | Directives | Implementations |
| Form | Narrative, philosophical | MUST/SHOULD/MUST NOT with placeholders | Concrete values, no placeholders |
| Example | "Single Source of Truth: Each piece..." | "MUST NOT duplicate information" | "MUST NOT duplicate information from existing business specs" |
| Placeholders | None | [LAYER], [CONCEPT], [PREFIX] | business, BUS, login |
| Assumption | L0 is pure | L1 is structured | L2 is compiled |

## Compilation Rules

**L0 Principle → L1 Directive transformation:**

| L0 Form | L1 Compiled Form |
|---------|------------------|
| Narrative principle | MUST/SHOULD/MUST NOT directive |
| Conceptual "what" | Operational "how" with placeholders |
| WHY explanation | Validation requirement |
| Philosophy | Procedural instruction |

**Examples:**

**L0:** "Single Source of Truth: Each piece of information exists in exactly one place"
**L1:** "MUST NOT duplicate information from existing specs—use Foundation Reference for same-layer or Implements/Enables for upstream"

**L0:** "Layer Independence: Each layer receives requirements from its own prompt file"
**L1:** "MUST read from `.github/prompts/smaqit.[LAYER].prompt.md` as sole source of requirements"

**L0:** "Self-Validating Agents: Agents validate their own output before declaring completion"
**L1:** "MUST validate output against completion criteria before finishing"

## Notes

This agent is the **second** of three Level Up maintenance agents:
- **Agent-L0** (completed) — Maintains principle purity
- **Agent-L1** (this task) — Compiles L0 principles to L1 directives
- **Agent-L2** (future) — Compiles L1 templates to L2 agents

Agent-L1 is the compilation bridge between philosophy (L0) and implementation (L2). It transforms "why" into "how" while maintaining abstraction through placeholders.
