# Create Agent-L2: Agent Compiler

**Status:** Completed  
**Created:** 2026-01-11  
**Completed:** 2026-01-11  
**Parent:** B001 (Extensible Meta-Framework: Level Up Architecture)  
**Priority:** High

## Goal

Create the **forward-looking** Level 2 Agent Compiler agent (`smaqit.L2`) that compiles Level 1 template directives into Level 2 agent implementations. This agent maintains agent purity by ensuring concrete implementations properly compile from L1 directives without placeholders.

## Context

**What this is NOT:** A retroactive agent cleanup agent

**What this IS:** A compilation agent that:
- Compiles L1 directives with placeholders into L2 agents with concrete values
- Validates new agent implementations trace to L1 directives
- Maintains agent structure and self-containment
- Guides users on what belongs at L2 vs L0/L1

**Philosophy:** The agent assumes Level 2 agents are properly structured and maintains compilation discipline going forward through evolutionary development.

## Agent Responsibilities

### Input Scenarios

**Scenario 1: Compile Template to Agent**
- User: "Compile specification template to business agent"
- Agent: Populates placeholders ([LAYER]→business, [LAYER_PREFIX]→BUS, etc.) to generate concrete agent

**Scenario 2: Enhance Agent with Missing Implementation**
- User: "Add Layer Independence directives to business agent"
- Agent: Compiles L1 directive template into agent-specific MUST/MUST NOT statements with concrete values

**Scenario 3: L1 Contamination Prevention**
- User: "Add to business agent: Read from smaqit.[LAYER].prompt.md"
- Agent: "This is L1 (template with placeholder). The L2 form is: 'Read from smaqit.business.prompt.md'"

**Scenario 4: L0 Contamination Prevention**
- User: "Add to business agent: Layer Independence means each layer receives requirements"
- Agent: "This is L0 (principle). The L2 compiled form is: 'MUST read from .github/prompts/smaqit.business.prompt.md as sole source of requirements'"

### Core Behaviors

**Agent MUST:**
- Accept input about agent implementations (additions, enhancements, clarifications)
- Validate implementations compile from L1 directives
- Reject directive-form with placeholders (redirect to Agent-L1)
- Reject principle-form input (redirect to Agent-L0)
- Replace all placeholders with concrete values
- Preserve agent structure and consistency
- Guide users when they provide L0 philosophy or L1 templates

**Agent MUST NOT:**
- Accept narrative philosophy without compilation (that's L0)
- Accept directives with placeholders (that's L1)
- Add placeholders to agents ([LAYER], [CONCEPT], [PREFIX])
- Add principle explanations or rationale (belongs at L0)
- Modify L0 framework files (`framework/*.md`)
- Modify L1 templates (`templates/**/*.template.md`)
- Perform L0→L1 or L1→L0 transformations (that's Agent-L0/Agent-L1)

**Agent SHOULD:**
- Trace implementations to their L1 directive source
- Flag implementations with no clear L1 directive origin
- Maintain consistent implementation language across agents in same layer/phase
- Use concrete values for layer, phase, prefix
- Consolidate redundant implementations
- Ensure cross-references between agents remain consistent
- Embed necessary L1 directive content for self-containment

## Key Distinctions

| Aspect | Agent-L0 | Agent-L1 | Agent-L2 (this task) |
|--------|----------|----------|----------------------|
| Level | Principles | Directives | Implementations |
| Form | Narrative, philosophical | MUST/SHOULD/MUST NOT with placeholders | MUST/SHOULD/MUST NOT with concrete values |
| Example | "Single Source of Truth: Each piece..." | "MUST NOT duplicate information" | "MUST NOT duplicate information from existing business specs" |
| Placeholders | None | [LAYER], [CONCEPT], [PREFIX] | business, authentication, BUS |
| Values | Abstract concepts | Generic categories | Concrete layer/phase names |
| Assumption | L0 is pure | L1 is structured | L2 is compiled |

## Compilation Rules

**L1 Directive → L2 Implementation transformation:**

| L1 Form | L2 Compiled Form |
|---------|------------------|
| Directive with placeholder | Directive with concrete value |
| `[LAYER]` | `business`, `functional`, `stack`, `infrastructure`, `coverage` |
| `[LAYER_PREFIX]` | `BUS`, `FUN`, `STK`, `INF`, `COV` |
| `[PHASE]` | `development`, `deployment`, `validation` |
| `[CONCEPT]` | Concrete concept name (e.g., `login`, `authentication`) |

**Examples:**

**L1:** "MUST read from `.github/prompts/smaqit.[LAYER].prompt.md`"
**L2 (business agent):** "MUST read from `.github/prompts/smaqit.business.prompt.md`"

**L1:** "MUST use format `[LAYER_PREFIX]-[CONCEPT]-[NNN]`"
**L2 (functional agent):** "MUST use format `FUN-[CONCEPT]-[NNN]`"

**L1:** "MUST validate [LAYER_NAME] specification completeness"
**L2 (stack agent):** "MUST validate Stack specification completeness"

## Notes

This agent is the **third** of three Level Up maintenance agents:
- **Agent-L0** (completed) — Maintains principle purity
- **Agent-L1** (completed) — Compiles L0 principles to L1 directives
- **Agent-L2** (this task) — Compiles L1 templates to L2 agents

Agent-L2 completes the compilation pipeline from philosophy (L0) → directives (L1) → implementations (L2). It removes abstraction by replacing placeholders with concrete layer/phase-specific values, making agents self-contained and executable.
